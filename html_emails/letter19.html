Стабильная работа сайта зависит не только от хостинга и скорости вашего соединения. Когда пользователь в браузере отправляет запрос, DNS-сервер должен определить IP-адрес запрашиваемого сайта. В противном случае ресурс будет недоступен, даже при надёжной работе хостинга.
 

Избежать этого можно, если на DNS-запросы отвечают несколько географически распределенных серверов, и в случае выхода из строя одного из них, остальные продолжают работу, гарантируя доступность сайта из любой точки мира.
 

Подключите DNS-хостинг, чтобы повысить отказоустойчивость своих проектов — поддержка до 150 доменов, удобный редактор зон, API для создания собственных приложений.
